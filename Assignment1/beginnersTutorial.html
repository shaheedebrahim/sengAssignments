Beginners Tutorial

Where to get started?

Competitive programming can seem extremely daunting when first getting started there are dozens of algorithms, datastructures and techniques to learn. That being said if you break up the task of getting started into several subtasks it can make the process substantially easier.

This introduction will assume no knowledge of programming languages and will start from there

Steps to Getting Started

The steps to getting started can be broken up into several high level tasks

  1. Learn a programming language, I recommend learning the following (in order) for the reasons put in the next section:
    • Java
    • C++11
    • Python
  2. Once you have learnt at least one language put your skills to use by attempting an easy problem on Kattis the problems are organized by a difficulty level, try one that is less than 1.5. You can find more discussion about online judges by visiting the online judges page
  3. Now learn about asymptotic notation you can find a tutorial here: asymptotic notation
  4. Once you have done that take a few data structures and algorithms courses, I recommend algorithms 1 and algorithms 2 from Princenton University. They can be found here: Algorithms 1 and Algorithms 2
  5. Practice! The best way to learn now is to attempt real problems. You can find collections of problems with online judges all over the internet. A list of my favourites with their strengths and weaknesses can be found at here

Strengths of Each Language

Java is the most recommended due to its ease of use when first getting started, it also has a large library that can support a multitude of uses (BigInteger, BigDecimal etc). When using Java you will be able to solve almost any problem

Next is C++11, this is recommended as the next language as there are a few problems that Java will have troubles solving due to the Time Limit Exceeded error.

Last is Python as it well help to solve problems that can be quickly solved using the extensive Python library and also very easy problems that you can write quicker in the Python
